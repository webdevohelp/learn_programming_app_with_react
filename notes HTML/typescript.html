  1. In TypeScript, a type is an easy way to refer to the different properties & functions that a value assigned to any variable has. In TS whether you want it or not, but you are going to have a type with every value you assign.
    Types are used by compiler to analyze our code for possible errors.
    Types allow others to understand what values are flowing around our codebase.
  2. TypeScript has two different categories of types:
    Primitive Types: number, boolean, void, undefined, string, symbol, & null
    Object Types: functions, arrays, classes, & objects
  3. Type Annotations: Code we add to tell TS what type of a value a variable will refer to.
    Type Inference: TS tries to figure out what type of a vlaue a variable refers to.
