ReactJS & NextJS Notes


  1. React uses Declarative Approach. We define the desired target state(s) and let React figure out the actual JS DOM instructions.
  2. React-Events:
    React events are named using camelCase, rather than lowercase.
    With JSX you pass a function as the event handler, rather than a string.
  3. The first letter in the Component's name must be in caps when using in JSX.
  4. props.children:
    It is used to display whatever you include between the opening and closing tags when invoking a component.
  5. "Whatever" you write/declare inside component while using them can be accessed by "props.Whatever". In short, " Attributes are Props".
    Ex: <Component cat="dog" />
    value of cat can be accessed inside component using "props.cat"
  6. JSX is fundamentally syntactic sugar. After compilation, JSX expressions become regular JavaScript function calls and evaluate to JavaScript objects.
    We can embed any JavaScript expression in JSX by wrapping it in curly braces.
    Note: Only expressions, not statements; means directly we can not put any statement (if-else/switch/for) inside JSX.
  7. Following cannot be used in a JSX Expression:
    A for loop or while loop or any other loop
    A variable declaration
    A function declaration
    An if condition
    An object
  8. Rules of JSX:
    Return Single Element - Code must be wrapped in ONE top level element.
    div/section/article or React Fragment - to use as a parent element (wrapper).
    Use camelCase for HTML Attribute - ie. onClick
    className Instead of Class.
    Close Every Element, either self-closing or properly formatted.
    Formatting - while returning JSX, we must have parenthesis around JSX or place parent element's starting tag on same line with 'return'.
    Tags are elements - Use lowercase tags for DOM elements, and Capitalized tags for component elements.
  9. Each instance of the component will have it's own saperate "state". State is saperated on per component instanse basis.
  10. Any component can have multiple states and you can easily manage them saperately.

    If you update state that depends on the previous state, you should use the "function form" of the state updating function instead.
  12. Custom built components do not support "className" property in React. You have to apply the classes manually to custom built component.
